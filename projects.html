SkyDocu

×

SkyDocu is a newer version of DMS but with support for multiple containers and thus theoretically allowing running in cloud.

The application is divided into two parts: Superadministration and Containers.

Superadministration

Superadministration is used for container management, user management and background services management. Superadministrator can create containers here and see their usage statistics.

Container

Containers contain the whole DMS functionality. Users can manage documents, metadata, folders and processes here.

Each container has its own group. Membership in the container's group allows entering the container. This functionality means that each container has its own referent - the superadministrator that is responsible for the container.
However the referent doesn't need to be visible to administrators in the container.

As is it my most recent project (as of January 2025), it uses the newest version of my framework.

DMS (Document Management System)

×

Document Management System or DMS is a web application that allows users to manage their documents in one place.

It contains a lot of functionalities, i.e.: folders, custom metadata, sharing documents.

It is easily intalled and managed. It is created for on-premise use.

For this project I created my own framework. It allowed using AJAX and processing operations or requests in the background. My next projects utilize this framework or parts of it.

Topics

×

Topics was created as a light weight version of other websites that allow users to discuss topics.

It allows users to register and create topics (if they don't already exists) and post posts to the topics and discuss with other members.

It utilizes roles for easier management and finer rights management. The application itself has different roles that allow content regulation or other management.

The application itself partially shared codebase with my DMS project. In the beginning of this project I copied all the code from the DMS project and removed the whole UI and some of the API classes.
As I was building this project, I added to the framework new functions, updated the existing ones and removed the unused ones.
