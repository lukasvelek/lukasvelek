The framework I use in my newest projects

The framework currently has no name and is not available for usage in other projects. The reason for that is that I still haven't been able to separate it from the projects. However when I finish the separation, I'll link it here.

Disclaimer: The information found in this article is 100% relevant only to the SkyDocu project. Other projects using this framework might contain different functionalities.

The idea for creating my own framework began slightly after I joined the company where I am employed. Here we create web applications in PHP and we utilize Nette framework.
Ever since I started programming I never liked to use other people's code. Therefore most of my projects only utilized libraries like Bootstrap CSS or jQuery.
So, because I didn't want to learn Nette and its components, I decided to create my own framework that would follow the MVP.

Every application that uses this framework has "index.php" that is the point of all the action. In this script an instance of Application class is created.

Application class is the main class of the whole application. It calls the render engine that is responsible for further UI rendering. It contains instances of necessary repositories or managers.
It also has definitions of important functions.
This script is also responsible for displaying the content rendered by the Application class to the user.

UI

UI is divided into modules that allows separation of administration from the general UI.
Each module is further divided into presenters.
Every presenter can further describe more UIs. E.g.: LoginPresenter can be responsible for the login screen UI as well as the forgot password UI.
Each UI has its own template that must exist. This template can utilize variables:
$PAGE_TITLE$
If no definition for the variable exists it is displayed in its raw form -> what you see.

However templates can also define components:
$component COMPONENT_NAME$
Components must have a definition otherwise exception is displayed.

There are 4 types of functions available to the user in the presenters.
They are:

  • action<Action>()
  • handle<View/Action>()
  • render<View>()
  • createComponent<ComponentName>()

action<Action>() is called only in AJAX requests.
handle<View/Action>() is called before render<View>() and can handle form submit, user redirect, etc...
render<View>() is called after handle<View/Action>() and is intended for filling template variables.
createComponent<ComponentName>() is called only if the template contains a component. This method must return an instance of the IRenderable interface that "tags" the implementing class as renderable and calls the class' render() function.

Database operations

Every time the application is instantiated the connection to the database server is created. There usually is one single connection to the database.
Low-level API methods are defined in Repositories (classes that extend the ARepository abstract class). The repositories do not check rights.
High-level API methods are defined in Managers (classes that extend the AManager abstract class). The managers are intended for usage with right checking.
